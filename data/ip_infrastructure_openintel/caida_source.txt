Why these 16 snapshots
----------------------
The OpenINTEL IP-infrastructure datasets in ../OpenINTEL_sea_output cover the measurement
months 202503 through 202606 (16 months). CAIDA publishes one AS-org snapshot per month,
so ONE snapshot is downloaded per OpenINTEL measurement month, to map each row's ASN with
the CAIDA snapshot that is contemporaneous with that row's month (a longitudinal mapping,
not a single-snapshot approximation):

File format (pipe-delimited, two sections)
-------------------------------------------
  Org section:  org_id | changed | org_name | country | source
  AS  section:  aut(ASN) | changed | aut_name | org_id | opaque_id | source
An ASN is mapped to a provider by: ASN -> org_id (AS section) -> org_name (Org section),
with the short AS name (aut_name) as a fallback.
